Introduction

What is HTML

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the structrue of a webpage by using elements (tags) to define the different parts of a page such as heading, paragraphs, links, images, and more.

What is CSS

CSS (Cascading Style Sheets) is used to style and layout web pages. It controls the color, font, spacing, positioning, and responsiveness of HTML elements, ensuring an aesthetically pleasing and functional design.

How HTML and CSS Work Together

HTML structures the content, while CSS enhances its presentation. Without CSS, HTML pages appear as plain text and images. Combining them allows for visually engaging web pages with structured layouts and responsive designs.

HTML Basics

HTML Structure

Every HTML document has a basic structure.

                        
                        <!DOCTYPE html>
                        <html>
                            <head>
                                <title>Page Title</title>
                            </head>
                            <body>
                                <h1>This is a Heading</h1>
                                <p>This is a paragraph.</p>
                            </body>
                        </html>
  • <!DOCTYPE html> declares the document type
  • <html> is the root element of an HTML page
  • <head> contains meta information about the document

Common HTML Elements

  • <h1> - <h6> headings
  • <p> paragraphs
  • <a href="https://example.com">Click here</a> to create hyperlinks
  • <img src="image.jpg alt="Description"> images
  • <div> for layout sections
  • <span> for inline styling

Lists

In HTML, there are two main types of lists: unordered and ordered. The difference lies in how the items are presented. Here’s an explanation of each type:

Unordered List

An unordered list is used to display a list of items in no particular order. It is typically represented with bullet points.

  • <ul>: Defines an unordered list.
  • <li>: Defines each list item within the unordered list.

Here is an example of an unordered list:

                        
                        <ul>
                            <li>Item 1</li>
                            <li>Item 2</li>
                            <li>Item 3</li>
                        </ul>

Ordered List

An ordered list is used to display a list of items in a specific, numbered order. Each item is numbered by default.

  • <ol>: Defines an ordered list.
  • <li>: Defines each list item within the ordered list.

Here is an example of an ordered list:

                        
                        <ol>
                            <li>First</li>
                            <li>Second</li>
                            <li>Third</li>
                        </ol>

Tables

The <table> element is used to represent tabular data. It contains rows and columns to display data in a grid-like format. The following tags are typically used inside a table:

  • <table>: Defines the table itself.
  • <caption>: Provides a title or description for the table.
  • <thead>: Groups the header content in the table. It typically contains the column headings.
  • <tr>: Represents a row in the table.
  • <th>: Defines a header cell in the table. It’s usually bold and centered by default.
  • <tbody>: Groups the main content of the table (the body). It typically contains the rows of data.
  • <td>: Defines a cell of data inside the table rows.

Here is an example of a simple table:

                        
                        <table>
                            <caption>Sample Table</caption>
                            <thead>
                                <tr>
                                    <th>Header 1</th>
                                    <th>Header 2</th>
                                    <th>Header 3</th>
                                </tr>
                            </thead>
                            <tbody>
                                <tr>
                                    <td>Row 1, Cell 1</td>
                                    <td>Row 1, Cell 2</td>
                                    <td>Row 1, Cell 3</td>
                                </tr>
                                <tr>
                                    <td>Row 2, Cell 1</td>
                                    <td>Row 2, Cell 2</td>
                                    <td>Row 2, Cell 3</td>
                                </tr>
                            </tbody>
                        </table>

Forms

Forms are used to collect user input. They can contain various input elements such as text fields, checkboxes, radio buttons, and more. The <form> element is used to create a form, and the following tags are commonly used inside a form:

  • <form>: This tag defines the tag itself. It can include attributes such as action (the URL where the form data should be sent) and method (the HTTP method to use, such as GET or POST).
    • and method to specify where the form data should be sent and how it should be processed.
  • <input>: Represents an input field where users can enter data. The type attribute specifies the type of input, such as text, password, checkbox, radio, etc.
  • <label>: Provides a label for an input field. It improves accessibility by associating the label with the corresponding input field.
  • <textarea>: This tag creates a multi-line text input field, allowing users to enter larger amounts of text.
  • <button>: Creates a clickable button, which can be used to submit the form or trigger other actions.
  • <select>: Defines a drop-down list, allowing users to select one or more options from a list which are inserted using the <option> tag.

Here is an example of a simple form:

                        
                        <form action="/submit-form" method="post">
                            <label for="name">Name:</label>
                            <input type="text" id="name" name="name" required>
                            <label for="email">Email:</label>
                            <input type="email" id="email" name="email" required>
                            <label for="message">Message:</label>
                            <textarea id="message" name="message"></textarea>
                            <button type="submit">Submit</button>
                        </form>

Semantic HTML

Semantic HTML refers to the use of HTML markup that conveys meaning about the content contained within. By using semantic elements, developers can create more meaningful and accessible web pages. Semantic HTML helps search engines and assistive technologies understand the structure and context of the content, improving SEO and accessibility.

Some common semantic elements include:

  • <header>: Represents introductory content at the beginning of a section or page.
  • <nav>: Contains navigation links to other pages or sections of the same page.
  • <main>: Contains the main content of the page.
  • <article>: Represents an independent piece of content that can stand alone.
  • <section>: Defines a thematic grouping of content, typically with a heading.
  • <footer>: Contains footer information such as copyright, contact information, etc.
CSS Basics

CSS Syntax

CSS (Cascading Style Sheets) uses a syntax that consists of a selector and a declaration block. The selector targets the HTML element you want to style, and the declaration block contains the property-value pairs that define the style. Here’s an example of CSS syntax:

                        
                        selector {
                            property: value;
                        }

For example, to change the color of all <p> elements to red, you would write:

                        
                        p {
                            color: red;
                        }

Common CSS Properties

CSS properties define how HTML elements are displayed on a webpage. Here are some common CSS properties and their uses:

  • color: Sets the color of text.
  • font-size: Sets the size of the font.
  • background-color: Sets the background color of an element.
  • margin: Sets the margin around an element.
  • padding: Sets the padding inside an element.
  • border: Sets the border around an element.
  • display: Specifies how an element is displayed.
  • position: Specifies the positioning method of an element.

CSS Selectors

CSS selectors are used to target specific HTML elements for styling. There are several types of selectors, including:

  • Element Selector: Targets all instances of a specific HTML element, eg. h1 { color: blue; }
  • Class Selector: Targets elements with a specific class attribute, eg. .title { font-size: 24px; }
  • ID Selector: Targets an element with a specific ID attribute, eg. #header { background-color: black; }
  • Universal Selector: Targets all elements on a page, eg. * { margin: 0; }
  • Descendant Selector: Targets an element that is a descendant of another element, eg. div p { color: red; }
  • Pseudo-class Selector: Targets elements based on their state or position, eg. a:hover { text-decoration: underline; }
  • Pseudo-element Selector: Targets specific parts of an element, eg. p::first-line { font-weight: bold; }
  • Attribute Selector: Targets elements based on their attributes, eg. input[type="text"] { border: 1px solid black; }
  • Grouping Selector: Targets multiple selectors at once, eg. h1, h2, h3 { color: blue; }
  • Child Selector: Targets an element that is a direct child of another element, eg. ul > li { list-style-type: none; }

CSS Units

CSS uses various units for measurements, including:

  • px: Pixels, an absolute unit.
  • em: Relative to the font-size of the element.
  • rem: Relative to the font-size of the root element.
  • %: Percentage of the parent element.
  • vh and vw: Relative to the viewport height and width.

CSS Colors

Colors in CSS can be defined using:

  • Named colors (e.g., red)
  • Hexadecimal (e.g., #ff0000)
  • RGB (e.g., rgb(255, 0, 0))
  • RGBA (e.g., rgba(255, 0, 0, 0.5))
  • HSL (e.g., hsl(0, 100%, 50%))

CSS Box Model

The CSS box model is a fundamental concept in web design that defines the structure of elements on a webpage. It consists of four parts: content, padding, border, and margin. Understanding the box model is essential for creating layouts and styling elements effectively.

  • Content: The actual content of the element, such as text or images.
  • Padding: The space between the content and the border.
  • Border: The border around the padding and content.
  • Margin: The space between the border and other elements on the page.

Here is an example of the box model:

                        
                        .box {
                            width: 200px;
                            height: 200px;
                            padding: 20px;
                            border: 1px solid black;
                            margin: 10px;
                        }
CSS Layout and Positioning

CSS Layout Techniques

CSS provides several techniques for creating layouts on web pages. Two of the most popular methods are Flexbox and CSS Grid.

Flexbox

Flexbox (Flexible Box Layout) is a one-dimensional layout model that allows you to design a layout structure in a more efficient way than traditional methods. It is particularly useful for aligning items in a row or column.

Here’s an example of a simple navigation bar using Flexbox:

                        
                        .nav-container {
                            display: flex;
                            justify-content: space-around; /* Distributes space evenly between items */
                            align-items: center; /* Centers items vertically */
                            background-color: #333;
                            padding: 10px;
                        }
            
                        .nav-item {
                            color: white;
                            text-decoration: none;
                            padding: 10px 20px;
                        }
            
                        .nav-item:hover {
                            background-color: #555; /* Changes background on hover */
                        }

HTML structure for the navigation bar:

                        
                        <div class="nav-container">
                            <a href="#" class="nav-item">Home</a>
                            <a href="#" class="nav-item">About</a>
                            <a href="#" class="nav-item">Services</a>
                            <a href="#" class="nav-item">Contact</a>
                        </div>

CSS Grid

CSS Grid Layout is a two-dimensional layout system that allows you to create complex layouts easily. It enables you to define rows and columns, and place items within that grid.

Here’s an example of a simple photo gallery using CSS Grid:

                        
                        .gallery-container {
                            display: grid;
                            grid-template-columns: repeat(auto-fill, minmax(150px, 1fr)); /* Creates responsive columns */
                            grid-gap: 10px; /* Space between grid items */
                        }
            
                        .gallery-item {
                            background-color: #eee;
                            padding: 20px;
                            text-align: center;
                            border: 1px solid #ccc;
                        }

HTML structure for the photo gallery:

                        
                        <div class="gallery-container">
                            <div class="gallery-item">Image 1</div>
                            <div class="gallery-item">Image 2</div>
                            <div class="gallery-item">Image 3</div>
                            <div class="gallery-item">Image 4</div>
                            <div class="gallery-item">Image 5</div>
                            <div class="gallery-item">Image 6</div>
                        </div>

CSS Positioning

CSS positioning allows you to control the placement of elements on a webpage. There are several positioning methods:

  • Static: The default positioning; elements are positioned according to the normal flow of the document.
  • Relative: Positioned relative to its normal position. You can use the properties top, right, bottom, and left to adjust its position.
  • Absolute: Positioned relative to the nearest positioned ancestor (not static). It is removed from the normal document flow.
  • Fixed: Positioned relative to the viewport, meaning it stays in the same place even when the page is scrolled.
  • Sticky: A hybrid of relative and fixed positioning. It toggles between relative and fixed, depending on the scroll position.
Responsive Design and Media Queries

Responsive Design

Responsive design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. The goal is to ensure that users have a good viewing experience, regardless of the device they are using.

To achieve responsive design, it is important to include the following meta tag in the <head> section of your HTML document:

                        
                        <meta name="viewport" content="width=device-width, initial-scale=1.0">

This meta tag sets the viewport width to the device's width and sets the initial zoom level to 1.0, ensuring that your layout is responsive on mobile devices.

Responsive design can be achieved through flexible grids, layouts, images, and CSS media queries.

Media Queries

Media queries are a powerful feature of CSS that allow you to apply styles based on the characteristics of the device, such as its width, height, orientation, and resolution. This enables you to create responsive designs that adapt to different screen sizes.

The basic syntax of a media query is as follows:

                        
                        @media (condition) {
                            /* CSS rules here */
                        }

For example, to change the background color of a page for screens smaller than 600 pixels, you would write:

                        
                        @media (max-width: 600px) {
                            body {
                                background-color: lightblue;
                            }
                        }

You can also combine multiple conditions:

                        
                        @media (min-width: 600px) and (max-width: 1200px) {
                            body {
                                background-color: lightgreen;
                            }
                        }

Using media queries, you can adjust styles for different devices, ensuring a consistent and user-friendly experience across all platforms.
